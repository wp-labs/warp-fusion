# Self-Update Manifests

This directory stores release manifests for Warp Fusion update checks.

- `stable/manifest.json`: latest stable release
- `alpha/manifest.json`: latest alpha release
- `beta/manifest.json`: latest beta release

Notes:
- `sha256` is auto-generated by `.github/workflows/release.yml` from uploaded release artifacts.
- Local/manual testing can still use placeholders, but update install will fail until real checksums are written.
- Keep `assets` keys aligned with runtime target triples.
- Git tags do not contain the finalized manifest. A tag points to the commit
  that triggered the release; the workflow writes the completed manifest back
  to the channel branch after release assets exist.
- `manifest_base_url` is channel-specific by branch:
  - stable: `https://raw.githubusercontent.com/wp-labs/warp-fusion/main/updates/stable`
  - alpha: `https://raw.githubusercontent.com/wp-labs/warp-fusion/alpha/updates/alpha`
  - beta: `https://raw.githubusercontent.com/wp-labs/warp-fusion/beta/updates/beta`
